# Agent Setup Service — Overview

## What We Do

We set up AI-augmented development environments for software teams. Every developer gets an AI assistant that already knows the codebase, the conventions, the architecture decisions, and the constraints — from day one.

## Who It's For

- **Software agencies** managing multiple client projects with rotating developers
- **Product teams** scaling engineering while maintaining code quality
- **Companies adopting AI coding tools** that want more than generic autocomplete

## The Core Problem

AI coding tools (Copilot, Cursor, ChatGPT) generate code — but they don't know YOUR code. They don't know your naming conventions, your architecture decisions, your deployment constraints, or your client's compliance requirements. The result: developers spend time fixing AI suggestions instead of shipping.

## What Makes This Different

We don't just install tools. We build a **knowledge layer** on top of your existing repos:

1. **Per-project AI configuration** — coding standards, patterns, gotchas, architecture decisions
2. **Agency/org-wide standards** — shared conventions that apply across all projects
3. **Knowledge Dashboard** — web UI for visibility, search, and management
4. **Custom workflow commands** — automated code review, PR creation, testing, onboarding
5. **Role-based guardrails** — different levels of guidance for junior vs. senior developers

Everything is backed by git. No new infrastructure. No vendor lock-in. If you stop working with us, the knowledge stays in your repos.

## Service Model

| Phase | What Happens | Timeline |
|---|---|---|
| Discovery | Audit your workflow, stack, pain points | 1 session (1-2 hrs) |
| Setup | Knowledge files, commands, dashboard, training | 1-2 weeks |
| Tuning | Monitor usage, optimize configurations | Weeks 3-6 |
| Maintenance | Monthly updates, new projects, onboarding | Ongoing |

## Key Outcomes

- Context switching between projects drops from hours to minutes
- New developer onboarding drops from weeks to days
- Code review cycles cut in half
- Zero knowledge loss when developers rotate or leave
- Client-specific constraints enforced automatically, not manually
